The Code
function luhn(cardNumber) {
  // 1. Strip spaces/dashes, split into digits
  const digits = cardNumber
    .replace(/\D/g, '')
    .split('')
    .map(Number)
    .reverse();  // ← rightmost = index 0

  let sum = 0;

  for (let i = 0; i < digits.length; i++) {
    let d = digits[i];

    // 2. Every second digit (skip index 0) gets doubled
    if (i % 2 === 1) {
      d = d * 2;

      // 3. If > 9, subtract 9 (same as summing both digits)
      if (d > 9) d = d - 9;
    }

    // 4. Add to running total
    sum = sum + d;
  }

  // 5. Valid only if total divides evenly by 10
  return sum % 10 === 0;
}
Plain English
  1. 01
    Read right to left
    Reverse the digits so the rightmost (the "check digit") is always at position 0. This makes the position math simple — even indexes are untouched, odd indexes get doubled.
  2. 02
    Skip the check digit
    Index 0 — the last digit of the original number — is the check digit. It was chosen by whoever issued the card specifically so the whole sum lands on a multiple of 10. Never transform it.
  3. 03
    Double every second digit
    Indexes 1, 3, 5… (every other, starting from the second) get multiplied by 2. This alternating pattern is what gives the algorithm its error-detection power — a single wrong digit almost always changes the total.
  4. 04
    Tame large doubles
    If doubling gives 10 or more, subtract 9. This is identical to adding the two digits together — 7×2=14, and 1+4=5, and 14−9=5. It keeps every value single-digit.
  5. 05
    Sum everything
    Add all processed digits together — both the transformed and the untouched ones — into a single running total. This is the number you see counting up in the animation.
  6. 06
    Check mod 10
    If the total is exactly divisible by 10 (remainder = 0), the number is VALID. Any other remainder means INVALID. Change a single digit anywhere and the sum shifts — the check fails.
